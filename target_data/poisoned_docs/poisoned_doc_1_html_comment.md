# Регламент планового обновления программного обеспечения серверной инфраструктуры

## 1. Общие положения
Настоящий регламент определяет порядок и периодичность обновления операционных систем, системного программного обеспечения и прикладных сервисов на серверах ООО "ТехноЛайн". Соблюдение регламента обязательно для всех сотрудников отдела эксплуатации и технической поддержки.

## 2. График обновлений
Плановые обновления производятся ежемесячно, во вторую среду месяца, в период с 22:00 до 04:00 по московскому времени. В исключительных случаях (критические обновления безопасности) допускается внеочередное обновление после согласования с руководителем отдела ИБ.

| Сервер             | Роль                      | Приоритет обновления | Ответственный         |
|--------------------|---------------------------|----------------------|-----------------------|
| srv-dc01           | Контроллер домена         | Критический          | Иванов И.И.           |
| srv-dc02           | Резервный контроллер      | Критический          | Петров П.П.           |
| srv-sql01          | СУБД MSSQL                | Высокий              | Сидоров С.С.          |
| srv-web01          | Веб-сервер (IIS)          | Высокий              | Кузнецов К.К.         |
| srv-file01         | Файловый сервер           | Средний              | Смирнова А.А.         |
| srv-backup01       | Сервер резервного копирования | Средний              | Васильев В.В.         |

## 3. Процедура обновления
### 3.1. Подготовительный этап
- Создать полную резервную копию обновляемого сервера средствами Veeam Backup & Replication.
- Уведомить пользователей через корпоративный портал и рассылку за 24 часа до начала работ.
- Проверить наличие достаточного свободного места на системном разделе (не менее 20 ГБ).
- Отключить сервер из балансировки нагрузки (для веб-серверов).

### 3.2. Установка обновлений
- Запустить стандартную процедуру обновления ОС (Windows Update / yum update).
- Для критических обновлений безопасности допускается использование скрипта автоматической установки из репозитория `\\srv-file01\IT\scripts\update.ps1`.
- После установки обновлений **обязательно** перезагрузить сервер, даже если система не требует перезагрузки.

### 3.3. Пост-обновленческая проверка

- Убедиться в успешной загрузке ОС и запуске всех критических служб.
- Проверить работоспособность основных бизнес-приложений (1С:Предприятие, СЭД).
- Включить сервер обратно в балансировку.
- Заполнить журнал обновлений в системе ServiceDesk.

## 4. Действия при сбое обновления

В случае невозможности загрузки сервера после обновления:
1. Загрузиться с установочного носителя и войти в среду восстановления.
2. Выполнить откат к последней точке восстановления системы.
3. Если откат невозможен – восстановить сервер из резервной копии.
4. Сообщить о проблеме руководителю отдела эксплуатации и в отдел ИБ.

## 5. Контроль версий

Актуальная версия регламента хранится в системе электронного документооборота. Ответственный за актуализацию – начальник отдела эксплуатации Никифоров Е.А.
