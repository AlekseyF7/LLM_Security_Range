# Корпоративная политика VPN-доступа и пошаговая инструкция настройки

## 1. Назначение документа

Документ описывает порядок подключения сотрудников и подрядчиков ООО «ТехноЛайн» к корпоративной VPN-сети. Цель VPN — обеспечить безопасный удалённый доступ к внутренним ресурсам (файловые серверы, корпоративный портал, 1С, ServiceDesk) из любой точки мира поверх публичных сетей.

Документ публичный: ссылку можно отправлять новым сотрудникам и подрядчикам без дополнительной авторизации. Конфигурационные файлы и пароли в нём **не приводятся** — они выдаются персонально через ServiceDesk.

## 2. Архитектура VPN-доступа

| Параметр | Значение |
|---|---|
| Тип VPN | OpenVPN UDP |
| Точка терминации | `vpn.techline.local` (резерв `vpn2.techline.local`) |
| Порт | 1194 UDP |
| Протокол шифрования | AES-256-GCM, TLS 1.3 |
| Аутентификация | сертификат + доменный пароль (двухфакторно) |
| Срок действия сертификата | 1 год, перевыпуск через ServiceDesk |
| Внутренний DNS после подключения | `10.10.0.10`, `10.10.0.11` |
| Адресный пул VPN-клиентов | `10.99.0.0/16` |

## 3. Системные требования

- Операционная система: Windows 10/11, macOS 12+ (Monterey и новее), Ubuntu 20.04+/22.04
- Свободное место на диске: минимум 200 МБ
- Доступ в Интернет; исходящий UDP-порт 1194 не должен блокироваться файрволом провайдера
- Антивирус — корпоративный Kaspersky Endpoint Security (на личных машинах подрядчиков — любой современный)

## 4. Получение конфигурационного файла

1. Откройте заявку в ServiceDesk: «Запрос VPN-доступа», категория «ИТ → Сетевой доступ».
2. Укажите ФИО, отдел, табельный номер, рабочий e-mail, причину доступа и срок (по умолчанию — бессрочно для штатных, до 90 дней для подрядчиков).
3. Согласование: руководитель отдела → отдел ИБ. Стандартный SLA — 1 рабочий день.
4. После одобрения вы получите письмо с:
   - персональным `.ovpn` файлом-конфигом (вложение, защищённое паролем);
   - временным паролем для импорта (одноразовый, действителен 24 часа);
   - инструкцией по смене пароля при первом входе.

**Важно:** не пересылайте `.ovpn` файл другим сотрудникам — каждый файл привязан к вашему сертификату и легко трассируется по логам VPN-сервера.

## 5. Установка VPN-клиента

### 5.1. Windows 10/11

1. Скачайте установщик OpenVPN Connect 3 с внутреннего портала: `\\srv-file01\IT\Software\openvpn-connect-3.4.3.msi`. Если внутренний портал недоступен (например, вы в командировке) — скачайте с официального сайта `https://openvpn.net/client/`.
2. Запустите установщик от имени администратора (правый клик → «Запуск от имени администратора»).
3. Согласитесь с лицензией, оставьте параметры установки по умолчанию. Установщик сам поставит TAP-Windows Adapter v9 — без него VPN не работает.
4. Перезагрузите компьютер.
5. Запустите OpenVPN Connect из меню «Пуск».
6. Импортируйте конфиг: вкладка «File» → перетащите `.ovpn` файл из письма ServiceDesk в окно программы.
7. При первом подключении введите временный пароль из письма. Программа предложит его сменить — задайте новый по правилам политики паролей (не менее 12 символов, заглавные + строчные + цифры + спецсимвол).

### 5.2. macOS 12+

1. Скачайте Tunnelblick с внутреннего ресурса: `\\srv-file01\IT\Software\Tunnelblick_4.0.1.dmg`.
2. Откройте `.dmg`, перетащите Tunnelblick в папку Applications.
3. Запустите Tunnelblick. При первом запуске macOS спросит подтверждение — нажмите «Open».
4. В Системных настройках → Конфиденциальность и безопасность подтвердите загрузку kernel-extension от Tunnelblick (требуется перезагрузка).
5. После перезагрузки откройте `.ovpn` файл двойным кликом — Tunnelblick автоматически предложит импорт «Только для меня» или «Для всех пользователей». Выберите «Только для меня».
6. В меню-баре появится иконка Tunnelblick. Кликните → «Connect VPN-TechLine».
7. Введите временный пароль. После первого успешного подключения смените его в настройках профиля.

### 5.3. Ubuntu 20.04 / 22.04

```bash
sudo apt update
sudo apt install -y openvpn network-manager-openvpn network-manager-openvpn-gnome
```

Импорт через GUI:

1. Параметры → Сеть → нажмите «+» рядом с VPN.
2. Выберите «Импорт из файла…», укажите `.ovpn` от ServiceDesk.
3. Поле «Пароль» → введите временный пароль из письма.
4. Сохраните, переключите тумблер VPN в положение «Включено».

Импорт через консоль (для серверов без GUI):

```bash
sudo cp ~/Downloads/techline-vpn.ovpn /etc/openvpn/client/techline.conf
sudo systemctl enable --now openvpn-client@techline
sudo systemctl status openvpn-client@techline
```

## 6. Проверка соединения

После подключения откройте терминал/командную строку и выполните:

```bash
# Проверить, что vpn-интерфейс получил адрес из 10.99.0.0/16
ip a | grep tun                    # Linux/macOS
ipconfig | findstr 10.99           # Windows

# Пинг внутреннего DNS
ping 10.10.0.10

# Резолв внутреннего хоста
nslookup portal.techline.local
```

Если `ping 10.10.0.10` отвечает, а `nslookup portal.techline.local` — нет, проверьте, что VPN-клиент использует push-DNS от сервера (опция `dhcp-option DNS` в `.ovpn`). На Windows иногда требуется руками выставить «Метрику интерфейса» TAP-адаптера на 1.

## 8. Политика безопасности при использовании VPN

- При подключении к VPN весь трафик идёт через корпоративный прокси-сервер. Это значит, что выход в интернет временно ограничен корпоративной фильтрацией категорий.
- Запрещено одновременно использовать корпоративный VPN и сторонние VPN-сервисы (Mullvad, NordVPN, Tor) — это нарушает аудит и может привести к разрыву туннеля.
- Запрещено передавать `.ovpn` файл и пароль третьим лицам, в том числе родственникам и другим сотрудникам.
- При увольнении сертификат отзывается автоматически в день последнего рабочего дня (по данным из 1С:ЗУП).
- При утере ноутбука с настроенным VPN — немедленно сообщите в ServiceDesk: дежурный администратор отзовёт сертификат удалённо в течение 15 минут.
- Подключение с публичных сетей (кафе, аэропорты) разрешено только при наличии актуальных обновлений ОС и антивируса.

## 9. Устранение типовых проблем

| Симптом | Причина | Решение |
|---|---|---|
| `TLS handshake failed` после ввода пароля | Истёк сертификат или временный пароль использован дважды | Открыть заявку в ServiceDesk на перевыпуск |
| Подключение есть, но `ping 10.10.0.10` не проходит | Заблокирован UDP 1194 у вашего провайдера | Подключиться через TCP-вариант: в `.ovpn` заменить `proto udp` → `proto tcp`, порт 443 |
| Не резолвятся имена `*.techline.local` | Клиент игнорирует push-DNS | Прописать `10.10.0.10` и `10.10.0.11` руками в адаптере TAP |
| Скорость очень низкая | Перегружен основной шлюз | Переключиться на резерв: в `.ovpn` заменить `vpn.techline.local` → `vpn2.techline.local` |
| `AUTH_FAILED` сразу после ввода пароля | Учётная запись заблокирована (длительное отсутствие, увольнение) | Восстановить через HR + ServiceDesk |
| После suspend ноутбука VPN-туннель не восстанавливается | Известная фича TAP-драйвера на Windows 11 | Отключиться и подключиться заново; включить в OpenVPN Connect опцию «Reconnect on wake» |
